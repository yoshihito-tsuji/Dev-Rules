# エスカレーション手順ガイド

Opus・Sonnet 間で解決しない技術的難問を Codex に相談する手順。

## いつ使うか

Opus・Sonnet 間で **2往復以内に解決しない技術的難問**が発生した場合。

## Opus 側の手順

1. Sonnet に「エスカレーション中: [問題の概要]」と送信して待機を伝える
2. Codex のセッションを別途開き、以下の形式で質問する:

   ```
   [エスカレーション相談]
   プロジェクト: [プロジェクト名]
   問題: [何が起きているか・何を決めたいか（1〜3行）]
   試したこと: [試みた解決策（1〜3行）]
   知りたいこと: [具体的な質問（1〜3行）]
   ```

3. Codex の回答をもとに設計判断を確定する
4. Sonnet に「エスカレーション完了: [結論]+[修正仕様]」を送信して再開する
5. Yoshihitoさんへ報告（事実・問題・結論を1メッセージで）

## xangi（LOCO）を使う場合

ユーザが検証・状況整理を行う場合に利用。Codex は技術判断役、xangi は対話・整理役として並列で活用できる。

## 注意事項

- エスカレーションは **Opus の権限**。Sonnet は自己判断でエスカレーションしない
- Codex への相談は3点（問題・試みた解決策・質問）のみで簡潔に
- 結論が出たら必ず Sonnet に再開メッセージを送ること

## STOP 後の作業再開

1. `show-dialog.sh --team <TEAM>` で中断前のメッセージを確認する
2. sonnet の「中断:」メッセージで完了済み/未完了を確認する
3. `git log` と `git status` で WIP コミットの有無を確認する
4. Opus が再度仕様メッセージを送信して再開する（未完了部分のみ）
